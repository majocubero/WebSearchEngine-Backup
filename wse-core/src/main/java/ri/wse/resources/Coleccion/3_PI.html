¿Qué es un puntero inteligente?

Ernesto Bascón, http://oopscenities.net
Contestado 23/10/2017 · El autor tiene 154 respuestas y 128,2k vistas de respuestas

Es una variable de stack que se comporta como puntero debido a la sobrecarga del operador ->.

La idea detrás de los punteros inteligentes yace en que, al ser variables de stack, el compilador se encarga de llamar a su destructor cuando la variable sale de ámbito (esta característica se denomina RAII). Gracias a eso y a que es posible sobrecargar el operador de asignación y crear constructores copia, el desarrollador del puntero inteligente puede implementar políticas de manejo de memoria sobre el objeto del que está apuntando.

Desde C++98 el programador puede crearse sus propios punteros inteligentes, pero desde C++11, la librería estándar de C++ viene con 3 punteros inteligentes: unique_ptr, shared_ptr y weak_ptr; que implementan diferentes políticas de manejo automático de memoria.

Gracias a estos tres tipos de punteros, es muy poco probable que el programador C++ necesite hacer manejo manual de memoria, como lo que se hace en C ó en versiones de C++ anteriores al estándar del 2011.

Actualizado 27/4/2018

En programación estructurada u orientada a objetos sería un mecanismo que encapsula el comportamiento de un puntero tradicional (un valor que indica una dirección de memoria) y que lo dota de alguna característica adicional.

En los lenguajes modernos normalmente no se suelen usar punteros tradicionales y las asignaciones de objetos se entienden como copias de referencias. Posteriormente un recolector de basura gestiona cuando un objeto ya no es necesario y ha de eliminarse. Este proceso es un ejemplo de la noción habitual de smart pointer.

En C++ por ejemplo, en la librería estándar existen objetos que encapsulan algo similar. Mediante conteo de referencias y aprovechando los destructores y el stack se puede saber cuantas referencias tenemos a un objeto y de tal forma eliminarlo cuando se deja de usar la ultima referencia.

Aparte de la aplicación trivial la idea no esta limitada a la gestión de memoria, aunque si tendría siempre bastante que ver o estaríamos hablando ya de otro concepto mucho mas abstracto.
